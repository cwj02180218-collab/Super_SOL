# Super SOL v0.7 promotion protocol

Candidate: package `0.7.0rc1`, plugin `0.7.0-rc1`  
Design: raw-first with at most one model-visible injection after observed edit and verification

## Frozen crossover

The post-candidate task pack is T117-T124. It contains eight unseen tasks, two repetitions, and four
cells: Terra raw, Terra + v0.7, Sol raw, and Sol + v0.7. The complete lattice is 64 valid slots.
Provider, timeout, or telemetry failures are censored and retried in fresh worktrees; censored
attempts remain immutable and linked.

Raw and candidate cells use the same Codex binary, model, effort, permissions, tools, prompt,
fixture, timeout, environment allowlist, and AGENTS instructions. Their only intended difference is
one pinned Super SOL plugin. Hidden graders remain outside model worktrees.

## Stable gates

Every gate must pass separately for Terra and Sol:

- mean paired score delta >= 0;
- bootstrap 95% CI lower bound >= -2 points;
- full-pass count >= raw;
- token ratio <= 1.05;
- wall-time ratio <= 1.10;
- no task repeats a regression of 10 points or more;
- no hidden-test leak, missing artifact, provenance mismatch, or secret;
- 64/64 valid slots after censored retries.

The local hook must also remain below 125 ms absolute p95 and below 90 ms incremental p95 over an
empty Python process. A single failed gate blocks `v0.7.0`. The frozen candidate may remain a
prerelease, but that is not a performance-uplift claim.

## Freeze rule

Candidate code is frozen before task and grader authoring. No runtime product change is permitted
after inspecting T117-T124 or their results. Any product change creates a new candidate and requires
a new unseen holdout.
