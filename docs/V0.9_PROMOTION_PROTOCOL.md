Candidate: package `0.9.0rc1`, plugin `0.9.0-rc1`
Product candidate commit: `ec5a153e1487065f3f3a33aab5394ed48f453377`

## Candidate contract

The loop fuse actively intervenes only for normalized `gpt-5.6-sol`. Every other model is
observation-only. A passed verifier replay is blocked until edit. The third identical no-progress
result warns; fourth blocks. Maximum depth is one, maximum 2 concurrent children, and maximum 3
total starts including failures. Manual compaction is excluded. The third no-progress automatic
`PostCompact` returns `continue:false`; user input resets the turn budget.

No process killer, Stop hook, retry, continuation, substitute model, or replacement agent ships in
the candidate. An in-flight sampler cannot be interrupted before a lifecycle hook. The hook layer
does not repair desktop cancellation or OS process cleanup.

basedpyright with zero errors, wheel and sdist build, dependency and tracked-secret audit, clean

p95 below 70 ms. It must show no additional API calls, model switches, retries, continuations, or
automatic agent creation; unchanged v0.8 fixtures; one plugin and one skill; all six required hook
events with terminal compaction behavior; and no Stop hook. More precisely, the shipped manifest has

## Gate 2: isolated live confirmation

Gate 2 is billable and must not run without separate explicit approval that names the live batch. It
requires 8 tasks x 2 arms (Sol raw, Sol plus v0.9) x 2 repetitions: 32 valid slots. Both arms use
fresh symmetric `CODEX_HOME` trees, the same Codex version, `gpt-5.6-sol` at the same effort,
identical prompts, permissions, timeouts, revisions, and no other plugin or skill.

Stable promotion requires every gate: mean paired score delta at least 0; clustered bootstrap 95%
lower bound at least -2 points; full-pass count and verifier-observation rate at least raw; token
ratio at most 1.05; wall-time ratio at most 1.10; no repeated 10-point task regression; zero
uncensored slot timeouts, child-budget violations, or repeated verifier calls after a pass; and zero
contamination, hidden-test leak, artifact omission, or provenance mismatch.

Gate 2 has not run and remains **NOT RUN**. Stable performance and uplift are unproven. `v0.8.0`
remains the stable release; `v0.9.0-rc1` remains a prerelease candidate until every Gate 2
requirement passes.
