# Super SOL v0.8 promotion protocol

Candidate: package `0.8.0rc1`, plugin `0.8.0-rc1`  
Scope: raw-first intervention only for normalized `gpt-5.6-sol` at reasoning effort `high`

## Candidate contract

Super SOL emits no prompt-time context. For Sol, one successful edit and the first recognized
verification result may produce at most one residual or repair context. Each context is at most 180
Unicode code points and names one boundary plus one observable check.

Terra, Luna, missing model metadata, malformed model metadata, and unknown model identifiers are
observation-only: no model-visible context, model switch, subprocess, subagent, retry, or additional
API call. Secret-shaped prompt blocking and billable-command authorization remain available. The
plugin makes zero extra model calls for every profile.

## Candidate Gate 0

Before sealed task-pack authoring, the candidate must pass the free gate: full and plugin-focused
test suites, at least 90% line coverage, clean Ruff and basedpyright, reproducible wheel and sdist
without forbidden runtime artifacts, a clean production dependency audit and tracked-secret scan,
an isolated one-plugin/one-skill lifecycle, and the 300-invocation hook p95 gates. Sol must emit at
most one context only after edit plus verification; every non-Sol profile must emit zero contexts.

## Stable-promotion gates

Every gate must pass:

- mean paired score delta is at least 0;
- clustered bootstrap 95% lower bound is at least -2 points;
- full-pass count is at least raw;
- token ratio is at most 1.03;
- wall-time ratio is at most 1.10;
- no task repeats a regression of 10 points or more;
- verifier-observation rate is at least raw;
- 96/96 valid slots exist after linked infrastructure retries;
- hidden-test leak, artifact omission, contamination, provenance mismatch, and secret count are zero.

Passing these gates permits the stable claim "noninferior quality with bounded overhead." A quality
uplift claim additionally requires a positive mean, a clustered bootstrap lower bound above zero,
and no full-pass regression. Otherwise uplift was not proven. These gates evaluate procedural
adherence and end-to-end agent performance, not underlying model intelligence.
They cannot show that the harness raises underlying model intelligence.

## Freeze rule

The candidate is frozen before T125-T136 and their private graders are authored. Sol/high validation
is pending until Gate 0 and every confirmatory gate are observed for the frozen commit. Any product
change after task-pack inspection creates a new candidate and requires a new unseen holdout.
