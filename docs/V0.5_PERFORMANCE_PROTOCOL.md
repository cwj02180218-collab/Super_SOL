# Super SOL v0.5 Performance Amplifier protocol

Date: 2026-07-12  
Candidate: `0.5.0rc1` / plugin `0.5.0-rc1`

## Question

Can GPT-5.6 Terra at medium effort with Super SOL v0.5 materially outperform Terra/medium raw and
finish within a fixed quality margin of GPT-5.6 Sol at high effort on bounded software-engineering
tasks?

This is a capability-tier comparison, not a claim that a plugin unlocks a ChatGPT Pro entitlement.

## Fixed cells

| Cell | Model | Effort | Plugin | Role |
|---|---|---|---|---|
| `terra-raw` | `gpt-5.6-terra` | medium | off | Plus-efficient baseline |
| `terra-v05` | `gpt-5.6-terra` | medium | v0.5 | amplifier candidate |
| `sol-high-raw` | `gpt-5.6-sol` | high | off | Pro-grade reference |
| `sol-max-raw` | `gpt-5.6-sol` | max | off | ceiling reference only |

The primary causal effect is `terra-v05 - terra-raw`. The model, effort, prompt, fixture, timeout,
permissions, grader, and environment are identical for those cells. Sol/max has no promotion gate.

## Runtime boundary

The plugin performs deterministic local routing. Generic, ambiguous, mixed, explanation-only, and
non-action prompts add no model context. A high-confidence request may receive exactly one frozen
procedure for concurrency, security boundaries, migration compatibility, or failure atomicity. A
recognized failed verification may produce one repair context inside the same turn.

The plugin never starts another Codex process, switches a model, calls an API, creates a subagent, or
purchases usage. `SUPER SOL OFF` on the first line forces pass-through for that turn. All live
benchmark commands remain separately gated by explicit approval and `--confirm-billable`.

## Gates

Gate 1 uses disclosed T102, T103, T104, and T112: four cells, two repetitions, 32 paid slots. It is
regression evidence only.

T109-T116 are disclosed and cannot be reused as an unseen holdout. Gate 2 requires externally supplied,
sealed T117-T124: four cells, two repetitions, 64 paid slots. The candidate cannot run Gate 2 unless
Gate 1 passes and an independent parser reproduces the evidence.

The fixed Gate 2 requirements are:

1. candidate amplification over Terra raw is at least **+5** mean score points;
2. the paired bootstrap 95% lower bound is at least 0;
3. candidate full-pass rate is greater than Terra raw;
4. candidate mean quality is no more than **-2** points below Sol/high raw;
5. no task repeats a regression of 10 points or more;
6. candidate/raw token ratio is at most **1.15**;
7. candidate/raw wall-time ratio is at most **1.15**;
8. pass-through token and wall-time ratios are each at most 1.05;
9. artifact omissions and leakage findings are zero;
10. the independent aggregate reproduction succeeds.

## Claim boundary

Before all gates pass, v0.5 is only a performance-amplifier candidate. Passing Gate 1 permits no
performance language. Even after a successful Gate 2, the project will report only the published
holdout result. It will not claim universal Pro equivalence, account-limit changes, or an intrinsic
model capability increase.
