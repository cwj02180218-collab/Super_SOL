# Super SOL v0.6 four-arm diagnostic protocol

Date: 2026-07-12  
Candidate: `0.6.0rc1` / plugin `0.6.0-rc1`

## Question

Did v0.5 fail because the router missed the right procedure, because the frozen procedure was
ineffective, or because Terra/medium reached a capability ceiling?

This protocol is **diagnostic-only** and permits **no performance promotion** or public uplift claim.

## Frozen arms

| Arm | Plugin behavior | Purpose |
|---|---|---|
| `terra-raw` | plugin absent | quality and resource baseline |
| `router-observe` | classify and record; inject nothing | routing precision and recall |
| `procedure-forced` | independent label forces one frozen pack | procedure effect without router misses |
| `adaptive-v06` | normal v0.6 routing and repair | end-to-end candidate |

All arms use the same Terra/medium model, prompt, fixture, timeout, permissions, and grader. Diagnostic
mode is supplied only through the local benchmark environment; it never changes the model-visible
prompt. Raw has no Super SOL skill.

## Disclosed development lattice

T109-T116 are disclosed and may be used only for diagnosis. Four arms across eight tasks and two
repetitions produce **64 slots**. There is no automatic retry and completed slots are immutable.

Independent task labels are prepared before execution. Router metrics use the natural route recorded
by `router-observe`; forced procedure labels are never derived from adaptive output.

The routing gates are:

- specialist recall at least **0.80**;
- specialist precision at least **0.85**;
- failure-atomicity recall at least **0.90**;
- no benchmark task ID, fixture noun, or answer string in router signals;
- pass-through token and time ratios at most 1.03.

The procedure diagnostic requires forced procedure mean uplift of at least +5 points and no repeated
10-point regression. Adaptive must recover at least 70% of the forced uplift with token and wall-time
ratios at most 1.15. These thresholds decide whether work continues; they are not release evidence.

## Later holdout

Only after the disclosed diagnosis passes may an independent author create sealed T125-T132 tasks.
That separate holdout requires a new preregistration, independent audit, and fresh billable approval.
Until then, v0.6 is an unproven diagnostic candidate.
