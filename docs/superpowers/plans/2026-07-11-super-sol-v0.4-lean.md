# Super SOL v0.4 Lean Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a v0.4.0-rc1 lean Super SOL candidate and a clean-room Codex A/B harness that can prove non-inferiority without making an unapproved model call.

**Architecture:** The installed plugin keeps its local billing and verification hooks but replaces broad model context with one exact contract-sweep sentence for action and debug prompts. A separate Python CLI builds symmetric ephemeral Codex homes, runs raw and lean arms through an injected process boundary, parses JSONL telemetry, grades workspaces out of band, and produces a fail-closed promotion report. Gate 0 exercises every path with fakes and `--dry-run`; quota-consuming Gate 1 and Gate 2 remain explicit later operations.

**Tech Stack:** Python 3.12, Pydantic 2, AnyIO 4, Typer, stock Codex CLI 0.144.1 or newer, pytest, Ruff, basedpyright, uv, existing digest-pinned Docker grader boundary.

## Global Constraints

- Stock Codex only; do not add LazyCodex, OMO, automatic subagents, or automatic continuations.
- Super SOL must never choose or change the model or reasoning effort.
- The exact action/debug context is `Silently map requirements to code and one boundary. After tests pass, re-read once for ownership, input, state, and failure semantics. Do not rerun tests.`
- `conversation` emits no model-visible context; `release` emits only its existing compact readiness context.
- Remove `SessionStart`; retain `UserPromptSubmit`, `PreToolUse`, `PostToolUse`, and `Stop`.
- Set `allow_implicit_invocation: false`.
- Dry-run and all Gate 0 verification must create zero Codex model processes.
- Live execution requires `--confirm-billable`; the plugin also requires the standalone user confirmation `SUPER SOL BILLABLE RUN APPROVED`.
- Both A/B arms use one absolute Codex binary, version, model, effort, prompt, task tree, permissions, timeout, environment allowlist, and command-flag tuple.
- Never use `--ignore-user-config` on only one arm, and never use `--dangerously-bypass-approvals-and-sandbox`.
- The only allowed home-tree difference is the digest-pinned Super SOL marketplace/plugin installation and generated local plugin state.
- Authentication may be copied to both ephemeral homes with mode `0600`, but its bytes and digest must never enter output artifacts; delete both homes after the run.
- Rate limits, session limits, timeouts, provider errors, telemetry errors, and grader infrastructure errors are missing infrastructure outcomes, never score zeroes.
- Do not publish a performance claim from T105-T108; only preregistered T109-T116 Gate 2 results may support a claim.
- v0.3.1 remains the immutable stable release; this implementation is `0.4.0rc1` in Python metadata and `0.4.0-rc1` in the Codex plugin manifest.

class HomeCommandRunner(Protocol):
    def run(self, argv: tuple[str, ...], env: Mapping[str, str]) -> CommandCapture: ...


class FakeHomeRunner:
    def __init__(self, plugin_source: Path, revision: str) -> None:
        self.plugin_source = plugin_source
        self.revision = revision
        self.calls: list[tuple[tuple[str, ...], dict[str, str]]] = []

    def run(self, argv: tuple[str, ...], env: Mapping[str, str]) -> CommandCapture:
        self.calls.append((argv, dict(env)))
        if argv[:3] == ("git", "-C", str(self.plugin_source)):
            if argv[3:] == ("status", "--porcelain"):
                return CommandCapture(0, "", "")
            return CommandCapture(0, f"{self.revision}\n", "")
        home = Path(env["CODEX_HOME"])
        if argv[1:4] == ("plugin", "marketplace", "add"):
            (home / "config.toml").write_text(
                '[marketplaces.super-sol]\nlast_updated = "2026-07-11T00:00:00Z"\n'
                f'source_type = "local"\nsource = "{self.plugin_source}"\n'
            )
            return CommandCapture(0, '{"marketplaceName":"super-sol"}', "")
        if argv[1:3] == ("plugin", "add"):
            with (home / "config.toml").open("a") as stream:

            return CommandCapture(0, '{"pluginId":"super-sol@super-sol"}', "")
        return CommandCapture(0, '{"installed":[{"name":"super-sol","enabled":true}]}', "")
