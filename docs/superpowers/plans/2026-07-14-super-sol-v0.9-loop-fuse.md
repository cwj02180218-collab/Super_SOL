# Super SOL v0.9 Loop Fuse Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Sol-gated, privacy-safe circuit breaker that bounds verifier replay, generic no-progress tool replay, recursive subagents, and ineffective automatic compaction while preserving v0.8 behavior.

**Architecture:** Keep `super_sol_hook.py` as a small dispatcher and split prompt routing, command normalization, bounded loop state, deterministic policy, and lifecycle adapters into focused modules. Every decision is derived from structured hook fields and a keyed, 4096-byte local ledger; no model, transcript parser, retry, continuation, or process killer is added.

**Tech Stack:** Python 3.12, stdlib-only plugin hooks, pytest 9, Ruff, basedpyright, Codex plugin JSON, uv, GitHub CLI.

## Global Constraints

- Active loop intervention is allowed only for normalized model `gpt-5.6-sol`.
- Terra, Luna, missing model metadata, malformed metadata, and unknown models are observation-only.
- A new `UserPromptSubmit` turn resets fuse state; internal work, compaction, and subagent messages do not.
- The v0.8 semantic residual and v0.9 soft warning share one model-visible soft-context claim.
- Every model-visible string is at most 180 Unicode code points.
- State stores no prompt, source, command, tool output, model output, path, environment value, or chat text.
- Each per-turn loop state file is at most 4096 bytes and retains at most eight action records.
- Default limits are depth 1, two concurrent children, three total children, warning on the third identical no-progress result, block on the fourth request, and stop on the third no-progress auto-compaction.
- No shipped `Stop` hook, automatic retry, model call, model switch, continuation, process kill, or replacement subagent.
- Production and test Python files touched or created by this plan must remain at most 250 non-comment, nonblank lines.
- Gate 2 is excluded from this implementation run because it is billable and requires a separate approval naming the live batch.

Use these exact public fields:

```python
from dataclasses import dataclass


@dataclass(frozen=True, slots=True)
class ActionRecord:
    fingerprint: str
    outcome: str
    streak: int
    edit_epoch: int


@dataclass(frozen=True, slots=True)
class LoopLedger:
    schema_version: int
    edit_epoch: int
    actions: tuple
    verifier_results: tuple
    active_agents: tuple
    pending_spawns: int
    total_agents: int
    compact_streak: int
    warned: bool
    terminal_reason: str | None
    last_progress_ns: int
    last_event_ns: int

    @classmethod
    def fresh(cls, now_ns: int = 0) -> "LoopLedger":
        return cls(1, 0, (), (), (), 0, 0, 0, False, None, now_ns, now_ns)
```

`keyed_fingerprint(root, raw)` must create one 32-byte installation key at
`root.parents[1]/.loop-key` using owner-only `O_EXCL`, then return the first 24 hexadecimal
characters of `hmac.digest(key, raw.encode(), "sha256")`. Never serialize `raw`.

most 200 ms, remove locks older than five seconds, load or reset state, apply the transition, reject
serialized output over 4096 bytes, atomically write `loop.json`, and release the lock in `finally`.
Corrupt and oversized ledgers are renamed to `loop.corrupt.<time_ns>` before returning fresh state.
Change `turn_root()` from the v0.8 `super-sol/v2` namespace to `super-sol/v3`; migration never reads
or copies a v0.8 turn ledger.
