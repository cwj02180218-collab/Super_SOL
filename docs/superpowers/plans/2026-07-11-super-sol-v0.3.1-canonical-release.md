> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Do not dispatch subagents for this repository.

**Goal:** Make `cwj02180218-collab/Super_SOL` the canonical standalone Super SOL repository and publish a verified v0.3.1 release with two accurate visual-guide assets.

**Architecture:** Preserve the existing Git history and contributor attribution, update only active product metadata and installation surfaces, and release a new immutable tag instead of rewriting v0.3.0. Render the infographic deterministically so every Korean character and evidence claim can be inspected before it is committed.

**Tech Stack:** Markdown, SVG/PNG assets, Python 3.12 package metadata, Codex plugin JSON, uv, pytest, Ruff, basedpyright, GitHub Actions, GitHub Releases.

- Canonical repository: `https://github.com/cwj02180218-collab/Super_SOL`.
- Release version: `0.3.1`; do not move or rewrite `v0.3.0`.
- Keep all existing commit authors and contributor history.
- Keep the plugin free of automatic billable API calls, model switching, and automatic subagents.
- Describe Super SOL as a harness and Codex plugin, not a model or ontology.
- Do not claim Fable parity, superior model intelligence, guaranteed security, zero total cost, or automatic performance improvement.
- Use stock Codex inline execution only; do not delegate.
- Administrator-only repository settings are reported separately if the current GitHub credential cannot change them.
