# Super SOL v0.7 Evidence-Bounded Router Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and release a raw-first Super SOL candidate that performs one bounded semantic
contract check after observed successful verification, then promote it only if a frozen 64-slot
holdout satisfies quality and cost gates.

**Architecture:** Replace v0.6 up-front adaptive packs with a deterministic contract router that is
silent in normal `UserPromptSubmit`. Record privacy-safe edit and verification events locally, then
emit at most one short residual or repair context from `PostToolUse`. Freeze the plugin candidate
before creating a separate T117-T124 holdout target and running the four-cell Orca crossover.

**Tech Stack:** Python 3.9-compatible Codex hooks, Python 3.12 package/tests, pytest, Ruff,
basedpyright, uv, Orca CLI, GitHub CLI, self-contained HTML visualization.

## Global Constraints

- Base commit is `3a5f6fdb06ae555753867f6819f4d91716aeb32d`.
- Candidate package version is `0.7.0rc1`; plugin version is `0.7.0-rc1`.
- Adaptive `UserPromptSubmit` emits no model-visible procedure context.
- A turn receives at most one model-visible injection total: residual or repair.
- Residual context is at most 220 Unicode code points.
- Automatic API/model calls, subagents, retries, test reruns, and effort/model changes are zero.
- Prompt, source, file names, command text, tool output, model output, environment values, and
  credentials are not stored.
- Stable promotion requires every gate in the approved design; failure publishes prerelease only.

### Task 2: Implement the one-injection evidence state machine

**Files:**
- Modify: `plugins/super-sol/hooks/super_sol_hook.py`
- Modify: `plugins/super-sol/hooks/super_sol_state.py`
- Modify: `tests/plugin/test_tool_hooks.py`
- Create: `tests/plugin/test_v07_state_machine.py`

**Interfaces:**
- Consumes: request metadata from Task 1 and Codex tool hook payloads.
- Produces: bounded immutable `edit` and `verification` events plus one claim named
  `model-visible-context`.

- [ ] **Step 1: Write failing state-machine tests**

  Cover these exact sequences:

  ```text
  action -> edit -> successful verification -> residual once
  action -> successful verification without edit -> silence
  action -> edit -> success -> second success -> silence
  action -> edit -> failed verification -> repair once -> later success -> silence
  action -> edit -> failed verification -> second failure -> silence
  pass-through/ambiguous/observe -> any tools -> silence
  corrupt or oversized state -> fail open with warning and no context
  ```

  Use `apply_patch`, `Edit`, and `Write` payload examples for edit recognition and shell-free
  verification examples already supported by `_verification_command`.

- [ ] **Step 2: Run the new tests and observe RED**

  Run: `uv run pytest -q tests/plugin/test_v07_state_machine.py tests/plugin/test_tool_hooks.py`

  Expected: failures because edits are not observed and successful verification emits no residual.

- [ ] **Step 3: Add bounded event helpers**

  In `super_sol_state.py`, add typed helpers that atomically record privacy-safe event kinds and
  compute turn evidence without storing tool names or commands. Reuse hashed `tool_use_id`, owner
  permissions, 4096-byte reads, and `claim_once`.

- [ ] **Step 4: Observe edit tools**

  Extend hooks configuration and dispatch so `PostToolUse` handles `apply_patch|Edit|Write|Bash`.
  For edit tools, write only `{kind: "edit", success: bool}`. For Bash, parse the command in memory,
  write only `{kind: "verification", success: bool}`, and discard command text.

- [ ] **Step 5: Emit one residual or repair context**

  On first successful verification after an edit, claim `model-visible-context` and emit
  `residual_context(primary_contract)`. On first failed verification after an edit, claim the same
  name and emit the short repair context. A claimed turn can never emit again.

- [ ] **Step 6: Run state-machine, security, and existing hook tests**

  Run:

  ```bash
  uv run pytest -q tests/plugin/test_v07_state_machine.py tests/plugin/test_tool_hooks.py \
    tests/plugin/test_prompt_hooks.py tests/harness/test_observed_hooks.py
  ```

  Expected: all pass and no stored fixture contains prompt, command, file, or output text.

- [ ] **Step 7: Commit**

  ```bash
  git add plugins/super-sol/hooks plugins/super-sol/hooks/hooks.json \
    tests/plugin/test_tool_hooks.py tests/plugin/test_v07_state_machine.py
  git commit -m "Add one-shot post-verification contract checks"
  ```
