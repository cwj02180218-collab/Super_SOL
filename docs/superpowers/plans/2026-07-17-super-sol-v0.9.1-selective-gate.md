# Super SOL v0.9.1 Selective Gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a `v0.9.1-rc1` candidate that injects one bounded specialist contract only for a high-confidence Sol or Terra route, emits one evidence reminder only after observed verification debt, and preserves the Sol-only loop fuse.

**Architecture:** Keep the existing stdlib-only Codex hook process and deterministic router. Add typed prompt/evidence claims and idempotent event recording in `super_sol_state.py`, activate natural routes in `super_sol_prompt_hook.py`, and make debt/repair/residual/loop warnings share one evidence channel. Freeze the product before authoring the separate confirmatory holdout.

**Tech Stack:** Python 3.12 project code, macOS system Python 3.9-compatible plugin hooks, pytest 9, Ruff, BasedPyright, uv, Codex plugin JSON, Git, GitHub CLI.

## Global Constraints

- Package candidate is `0.9.1rc1`; plugin and Git prerelease are `0.9.1-rc1`.
- Active semantic profiles are exact `gpt-5.6-sol` and exact `gpt-5.6-terra`; every other model is observation-only.
- The loop fuse remains active only for exact Sol.
- One `prompt` context and one shared `evidence` context are the absolute per-turn maximum.
- Every model-visible context is at most 180 Unicode code points.
- No extra model/API call, model switch, subagent, automatic retry, automatic continuation, or automatic passing-test rerun.
- No prompt, source, path, command, output, raw model string, API key, or secret is persisted.
- `v0.8.0` remains stable until the separate 240-slot holdout passes every profile gate.
- T201-T230 and every earlier task are diagnostic-only and cannot support an uplift claim.

In `super_sol_evidence_hook.py`, add:

```python
_ACTIVE_PROFILES = frozenset({"sol", "terra"})
_DEBT_EDIT_THRESHOLD = 2


def _model_visible_eligible(payload: dict[str, object], state: dict[str, object]) -> bool:
    profile = model_profile(payload)
    return (
        profile in _ACTIVE_PROFILES
        and state.get("model_profile") == profile
        and state.get("diagnostic_mode") != "observe"
    )


def _verification_debt(events: tuple[dict[str, object], ...]) -> bool:
    successful_edits = 0
    for event in events:
        if event.get("kind") == "verification":
            successful_edits = 0
        elif event.get("kind") == "edit" and event.get("success") is True:
            successful_edits += 1
    return successful_edits >= _DEBT_EDIT_THRESHOLD
```

is true and the profile is eligible, return `DEBT_CONTEXT` only when

evidence claim.
